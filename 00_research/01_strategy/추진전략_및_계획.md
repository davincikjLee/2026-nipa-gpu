## 3. GPU 활용 전략

### 3.1 GPU 용도 배분

| GPU | 용도 | 세부 내용 | 담당 |
|-----|------|----------|------|
| **B200 #1** | 로봇 제어코드 변환 LLM | 120B급 LLM Fine-tuning (gpt-oss, devstral-2, deepseek-r1) | 이형주 |
| **B200 #2** | 멀티모달 비전 모델 | 캘리브레이션/품질검사 VLM (llama3.2-vision 90B, qwen2.5vl 72B) | 이형주 |
| **B200 #3** | Isaac Sim 디지털트윈 | 로봇팔 공정 시뮬레이션, 강화학습 최적 경로 생성 | 이건주, 이범찬 |

### 3.2 B200 선택 근거

| 요구사항 | H100 한계 | B200 해결 |
|---------|----------|----------|
| **120B급 LLM 학습** | 80GB HBM3 부족 (gpt-oss 120B 불가) | **192GB HBM3e** |
| **90B VLM 동시 추론** | 메모리 부족 | **192GB로 다중 모델 로드** |
| **Isaac Sim + RL** | 시뮬레이션+학습 병렬 불가 | **8TB/s 대역폭** |
| **Omniverse 실시간** | 물리 연산 성능 부족 | **20+ PFLOPS FP8** |

**활용 예정 모델 메모리 요구사항**:

| 모델 | 파라미터 | FP16 메모리 | H100 (80GB) | B200 (192GB) |
|------|---------|------------|-------------|--------------|
| gpt-oss | 120B | ~240GB | ❌ 불가 | ✅ 가능 |
| devstral-2 | 123B | ~246GB | ❌ 불가 | ✅ 가능 |
| llama3.2-vision | 90B | ~180GB | ❌ 불가 | ✅ 가능 |
| qwen2.5vl | 72B | ~144GB | ❌ 불가 | ✅ 가능 |
| deepseek-r1 | 70B | ~140GB | ❌ 불가 | ✅ 가능 |

## 4. 연구 내용

### 4.1 Task 1: 로봇 제어코드 변환 모델 (GPU #1)

**목표**: 이기종 로봇 간 제어코드 자동 변환을 위한 대규모 언어모델 개발

| 세부 내용 | 설명 |
|----------|------|
| TDL 변환 모델 | 현대로보틱스 ↔ TDL ↔ 두산로보틱스 Jobfile 상호 변환 |
| 120B급 LLM Fine-tuning | gpt-oss(120B), devstral-2(123B), deepseek-r1(70B) 기반 |
| 다기종 확장 | 10개+ 로봇 제조사 통합 변환 모델 개발 |
| 강화학습 연동 | 최적 동작 계획 생성 (딥테크팁스 과제 연계) |

**기 확보 데이터**: 현대로보틱스 Jobfile, 두산로보틱스 Jobfile

**활용 모델**:

| 용도 | 모델 | 파라미터 |
|------|------|---------|
| 코드 변환 | gpt-oss | 120B |
| 코드 변환 | devstral-2 | 123B |
| 코드 변환/강화학습 | deepseek-r1 | 70B |
| 코드 변환 | qwen3-coder | 30B |

**B200 필수 이유**: 120B급 모델 Fine-tuning에 192GB HBM3e 메모리 필수

### 4.2 Task 2: 비전 기반 로봇 캘리브레이션 및 품질검사 (GPU #2)

**목표**: 멀티모달 비전 모델 기반 로봇 제어 오차 보정 및 품질 검사 시스템 개발

| 세부 내용 | 설명 |
|----------|------|
| 캘리브레이션 모델 | 비전 데이터 → 로봇 제어 미세 오차 보정 |
| 품질검사 모델 | 이미지 인식 → 정상/비정상 분류 |
| 실시간 멀티카메라 | 다중 시점 동시 분석 |
| 자율 보정 시스템 | 오차 감지 → 자동 캘리브레이션 |

**활용 모델**:

| 용도 | 모델 | 파라미터 |
|------|------|---------|
| 비전-언어 | llama3.2-vision | 90B |
| 비전-언어 | qwen2.5vl | 72B |
| 비전-언어 | qwen3-vl | 32B |
| 비전 특화 | DINOv2 (Meta) | - |

**B200 필수 이유**: 90B VLM + 72B VLM 동시 추론 시 H100 메모리 부족

### 4.3 Task 3: Isaac Sim 기반 로봇 공정 디지털트윈 (GPU #3)

**목표**: NVIDIA Omniverse Isaac Sim 기반 자동차 제작 로봇팔 공정 디지털트윈 구축

| 세부 내용 | 설명 |
|----------|------|
| 로봇팔 공정 모델링 | 자동차 제작 공정 로봇팔 작업 3D 모델링 |
| Jobfile 시각화 | 3D 환경에서 로봇 동작/잡파일 직관적 확인·수정 |
| 충돌/간섭 검증 | 충돌, 간섭, 작업 불가 자세 사전 검증 |
| 강화학습 환경 | Isaac Sim 기반 로봇 최적 경로 강화학습 |

**Isaac Sim 요구 사양** (4.2버전 기준):

| 항목 | 최소 | 권장 |
|------|------|------|
| GPU | RTX 3070 (8GB) | RTX 4080 (16GB) |
| RAM | 32GB | 64GB+ |

> **참고**: Isaac Sim은 일반 데스크톱 환경에서 구동 어려움 → B200 고성능 GPU 필수

**B200 필수 이유**: 물리 시뮬레이션 + 강화학습 + 대형 LLM 동시 구동

**벤치마크**: Foxconn Digital Twin (공장 배치 시간 50% 단축, 에너지 30% 절감)
