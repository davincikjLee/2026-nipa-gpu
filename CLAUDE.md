## AI 기반 사업계획서 작성 워크플로우

> 본 프로젝트에서 Claude와 함께 사용한 워크플로우입니다.
> 정부지원사업 신청 문서 작성 시 재사용 가능합니다.

### Phase 1: 공고 분석 및 구조화

```
입력: 공고문 PDF
출력: 공고_요약본.md

1. 공고문 PDF 업로드
2. Claude에게 핵심 내용 추출 요청:
   - 신청 자격
   - 지원 내용 (GPU 종류, 수량, 기간)
   - 평가 기준 (배점 포함)
   - 제출 서류
   - 일정
3. 00_research/00_notification/공고_요약본.md 생성
```

### Phase 2: 추진 전략 수립

```
입력: 공고_요약본.md + 기업 역량
출력: 추진전략_및_계획.md

1. 평가 기준 분석 → 핵심 포인트 도출
2. GPU 용도/필요성 구체화
3. 참여자 역할 분담
4. 연구 내용 (Task) 정의
5. 00_research/01_strategy/추진전략_및_계획.md 생성
   → 이 문서가 Single Source of Truth
```

### Phase 3: 근거 자료 조사

```
입력: 추진전략_및_계획.md
출력: 03_reference/*.md, B200_3대_필요성_근거.md

1. 웹 검색으로 기술 근거 수집:
   - GPU 성능 벤치마크
   - 모델 메모리 요구사항
   - 유사 사례 (글로벌 기업)
2. 03_reference/ 에 조사 자료 저장
3. 02_justification/B200_3대_필요성_근거.md 작성
```

### Phase 4: 교차 검증

```
입력: B200_3대_필요성_근거.md
출력: 검증_결과_보고서.md

1. 문서 내 주장 목록 추출
2. 각 주장 웹 검색으로 검증:
   - 공식 출처 확인
   - 수치 정확성 확인
3. 검증 결과 보고서 생성
4. 오류 발견 시 원본 문서 수정
```

### Phase 5: 실제 과제 내용 반영

```
입력: 기존 문서 + 실제 과제 계획 (PDF/내부 문서)
출력: 수정된 추진전략_및_계획.md, B200_3대_필요성_근거.md

1. 실제 수행할 과제 내용 확인:
   - 기 확보 데이터
   - 기존 인프라
   - 연계 과제
   - 활용 예정 모델
2. 추진전략 문서에 실제 내용 반영
3. GPU 필요성 문서에 구체적 모델/메모리 명시
4. 확장 방향 추가 (현재만으로 부족해 보일 경우)
```

### Phase 6: 최종 문서 작성

```
입력: 모든 research 문서
출력: 01_proposal/사업수행계획서.*

1. 추진전략_및_계획.md 기반 사업수행계획서 초안
2. B200_3대_필요성_근거.md 에서 기술적 근거 인용
3. 공고_요약본.md 의 평가 기준에 맞춰 검토
4. HWP/PDF 변환 및 최종 검토
```
